Undertaking fine-grained permissions — a complex, long-standing problem for enterprise and large organizations

Overview

Admins of large GitHub organizations wanted more granular control over access management for their organization’s members and collaborators. Basically, they asked for the discretionary ability to permit or disable certain actions or features per user — a fairly challenging technical and design problem.

Check out the case study (password-protected) to learn how we finally tackled the problem and shipped the beta feature in just 4 months. View the case study
